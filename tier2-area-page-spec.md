## 最終回答：peatbid tier2「勝てるエリアページ改善仕様」

### 1. 検索意図の分解と上位要件

「ウイスキー買取 ＋地域」は**行動直前のTransactionalクエリ**。地域語は「自分のエリアで、損せず・面倒なく売りたい」の意。1ページで以下5問に答え切ることが上位要件：

| サブ意図 | 問い | 満たす要素 |
|---|---|---|
| いくらで売れる | 山崎12年は今いくら | **銘柄別の最新実売相場**（peatbidの一次データ＝最大の武器） |
| どこに売る | 地元店？宅配？ | 業者比較（全国宅配中心＋実在確認できた地元店） |
| どう売る | 持込/宅配/出張 | 売却方法の手順（宅配主動線） |
| 高く売るには | 箱なしでも？ | 高価買取のコツ（外箱・ギャラ・状態・終売/限定） |
| 損しない | 騙されない？ | 実績・査定無料/キャンセル無料の事実明示 |

大手はドメイン権威で④⑤を、listicleは③④で勝っている。**peatbidが唯一持つ優位は①の精度（実売中央値の一次性）。①を信頼の起点に②〜⑤を地域文脈で肉付け**するのが勝ち筋。

### 2. 現状tier2の致命的ギャップ（断定）

1. **意図②③④⑤が丸ごと欠落** → ユーザーは相場だけ見て他社へ離脱。**送客＝収益動線が機能していない**。
2. **県名差し替え48ページ＝scaled content abuse / doorway pages 直撃**。インデックス削除・サイト全体評価毀損リスク。
3. **「○○県での買取相場」全カード同一文言**＝自動生成・無価値シグナル。
4. **E-E-A-Tゼロ**：データ出典・算出方法・更新日・運営者が無く、**最大の武器（一次相場）の信頼性が伝わっていない**。
5. **構造化データ無し**＋**明確な送客CTA無し**（内部リンクのみ）。

### 3. 勝てるエリアページ設計（必須セクション・順序つき）

設計思想：**①相場の一次性で信頼 → ⑤コツで滞在 → ③売り方で背中を押す → ④業者比較でCV**。地域情報は装飾でなく「薄コンテンツ回避の実質」として使う。

**0. メタ/H1**
- H1：「大分県のウイスキー買取｜実売相場とおすすめ業者【2026年最新】」
- title：`ウイスキー買取 大分｜Yahoo実売中央値で見る相場とおすすめ業者｜peatbid`
- リード：冒頭で「相場はYahoo!オークション過去180日の実落札中央値」と**一次性を宣言**＋**更新日明記**。

**1. 結論ボックス（above the fold）**
- 県の実情に基づく結論（例「大分は買取店舗が少なく宅配買取が最適」）＋第一CTA。県内主要都市名を自然に含む。

**2. 銘柄別 実売相場テーブル（差別化の心臓部）**
- 50銘柄を**「定価 → 実売中央値 → 買取目安（中央値×買取率レンジ）」**の3列で“いくらで売れるか”に直答。
- 中央値・**サンプル数・更新日・前回比↑↓**を表示、銘柄別サブページへ内部リンク。

**3. 高価買取のコツ**
- 外箱・ギャランティ・状態・終売/限定の4〜5項目。可能なら**peatbidデータで「箱あり/なしの中央値差○%」**を数値提示＝唯一無二。

**4. 売却方法の比較（宅配中心・出張・店頭）**
- 「手間・スピード・対応エリア・向く人」を表比較。**宅配を主動線**にし手順を1-2-3で図解。

**5. 県のウイスキー売却事情（ユニーク化セクション）**
- 県内の店頭選択肢の有無・宅配が向く理由を**公的データで裏が取れる事実のみ**で記述（§4の核）。

**6. おすすめ業者比較（CVセクション）**
- **全国対応の宅配買取業者**を比較表（査定料・送料・キャンセル料・入金スピード・対応エリア・特徴）。地元店は**実在確認できた場合のみ**事実併記。各行に送客CTA、`rel="sponsored"`。

**7. 売却手順ガイド（E-E-A-T: Experience）**
- 「申込→梱包→集荷→査定→入金」の実プロセス。

**8. FAQ**
- 「箱なしでも売れる？」「大分から集荷に来る？」など**県名入りの実質Q**。

**9. 運営者情報・データ方法論・更新履歴（E-E-A-T: Trust）**
- 中央値の定義・期間・データソース・運営者・最終更新日。

**E-E-A-T**：Experience＝実手順＋自社データ分析／Expertise＝相場算出方法論ページ／Authoritativeness＝一次データ提供元としての立ち位置を全ページ宣言／Trust＝出典・更新日・査定無料の事実・アフィリ開示。

**構造化データ（schema.org）**
- `BreadcrumbList`（全tier2必須）、`FAQPage`、`Article`＋author/datePublished/dateModified、`Organization`。
- 相場は`ItemList`/`Product`＋`AggregateOffer`（**実売値のみ。捏造禁止**）。
- `Dataset`は一次データの意味付けとして付与は可だが、**リッチリザルト化は期待薄**（過信しない）。
- `Review`/`AggregateRating`は**実体ある口コミが無い限り使わない**（スパム違反）。

### 4. 48ページのユニーク性担保（薄コンテンツ/ドアウェイ回避）

**テンプレ共通でよいが、データ・文章・推奨が県ごとに変わること。** Yahoo実売中央値の「地域文脈変換」が核。県ごとに必ず変える：

2. **県の売却環境の客観事実**：人口規模・県庁所在地・買取店舗の多寡など、**公的データで裏取り可能な事実のみ**。
3. **県内地名の自然な使用**（結論・FAQ・見出し）。
4. **各業者の当該県への対応可否**（事実確認したもの）。
5. **地名入り県別FAQ**。
6. **相場の動的差分**（更新日・前回比・サンプル数は時点×銘柄で固有値）。
